# Security Policy

## Supported Versions

The project is pre-1.0. Security fixes target the current `main` branch unless a maintainer says otherwise.

## Reporting a Vulnerability

Please report security issues privately to the repository maintainers. Do not open a public issue with exploit details, private keys, account data, or logs that contain personal information.

Useful reports include:

- affected version or commit
- operating system
- reproduction steps
- expected and actual behavior
- impact and suggested fix, if known

## Sensitive Data

Never attach `data/`, `data-2/`, `store.enc`, `account.json`, private keys, passwords, tokens, or production configuration to public issues.
