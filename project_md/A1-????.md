## 0. 总则（先读）

1. **小步 + 每步可回退**：一步只搬一个模块。搬完跑 `npm test`（语法/单测护栏）+ 手动冒烟，绿了再 `git commit`。出问题立刻 `git checkout -- electron/main.js electron/<新模块>.js` 回到上一步。
2. **行尾陷阱**：`main.js` 是**混合 CRLF**（约 1941 行 CRLF + 顶部少量 LF）。编辑器要**保留原行尾**（VS Code 右下角确认是 CRLF，别让 Prettier/格式化把全文件改成 LF），否则 `git diff` 会变成整文件、淹没真实改动。每步用 `git diff --stat` 确认只动了预期行数。
3. **校验留在主进程/领域层**，不要因为拆分把权限判断挪到前端。
4. **不改协议字节与加密逻辑**；这三个模块都不碰 wire 协议，纯属代码搬家 + 依赖注入。
5. **核心难点**：三个子系统有共享可变状态，且 tray↔window 相互调用。解法是**依赖注入 + 上下文对象**，`main.js` 仍是状态拥有者和装配者。

## 1. 共享可变状态盘点（要注入的东西）

`main.js` 顶部的 module-level 状态（搬出去的函数都依赖它们）：

| 状态 | 性质 | 谁会改 |
| -- | -- | -- |
| `mainWindow` | 可变，createWindow 赋值、closed 置 null | windows |
| `tray` | 可变，setupTray 赋值 | tray |
| `p2p` / `vault` / `ft` | 解锁/startP2P 后赋值 | 全部 |
| `runtime`（对象：minimizeToTray/notifyEnabled/notifyPreview/closeAction/dnd） | 引用共享 | tray、windows |
| `isQuitting` / `suppressTrayOnMinimize` | 可变标志 | windows、tray |
| `chatWindows`（Map） | 引用共享 | windows |
| `shareHosts/sharePending/shareDownloads/shareEarlyDownloads/shareXferMids` | share 状态 | sharespace-host |
| `appIconImg`（getAppIcon 缓存） | 缓存 | tray、windows |

**关键**：可变标量（`mainWindow`/`tray`/`isQuitting`）不能用值传递，要用 **getter/setter** 或挂在一个共享 `ctx` 对象上按引用访问。

---

## 2. 推荐 DI 模式：上下文对象 + 工厂函数

在 main.js 建一个 `ctx`，把"被多个模块共享/会变的东西"用访问器暴露：

```js
// main.js
const ctx = {
  // electron 原语（也可各模块自己 require）
  // 共享状态访问器（按引用/晚绑定，避免传到一半还没赋值）
  getMainWindow: () => mainWindow,
  setMainWindow: (w) => { mainWindow = w },
  getTray: () => tray,
  setTray: (t) => { tray = t },
  getVault: () => vault,
  getP2P: () => p2p,
  getFt: () => ft,
  runtime,                      // 对象，按引用共享，直接读写其字段
  chatWindows,                  // Map，按引用共享
  isQuitting: () => isQuitting,
  setQuitting: (v) => { isQuitting = v },
  sendToRenderer, getAppIcon, selfId, resolveDataDir,
  isDev, DEV_SERVER_URL,
}
```

每个模块导出一个工厂 `createXxx(ctx)`，返回 main 需要的函数集合。**tray↔window 相互调用通过 ctx 晚绑定**：先建两个模块对象，再把对方挂到 ctx 上（`ctx.tray = tray; ctx.windows = win`），这样 `showWindow()` 里调 `ctx.windows.createWindow()`、`createWindow` 的事件回调里调 `ctx.tray.updateTrayState()`，避免循环 require 和初始化顺序问题。

```js
const trayMod = createTray(ctx);   ctx.tray = trayMod
const winMod  = createWindows(ctx); ctx.windows = winMod
```

## 6. 每步通用验证清单

1. `node --check electron/main.js && node --check electron/<新模块>.js`（语法）。
2. `npm test`（16 用例必须仍全绿——它们覆盖 crypto/vault/p2p/sharespace/pathutil/pinned，不覆盖 main/tray/window，所以**别只靠它**）。
3. `git diff --stat`：确认**只**动了预期文件、**不是**整文件 diff（整文件=行尾被翻，需修）；确认无 NUL（`tr -cd '\000' < 文件 | wc -c` 应为 0）。
4. 手动冒烟（上面各步清单）。
5. 通过 → `git commit`；不过 → `git checkout -- <文件>` 回退该步重来。

## 7. 建议提交粒度

每步一个 commit：`windows.js extract` → `tray.js extract` → `sharespace-host orchestration extract`。这样任一退步只回退一个 commit，且 review 清晰。

## 附：为什么这步必须在真机做
沙箱是 Linux 无显示环境，`require('electron')` 拿不到 `BrowserWindow/Tray/nativeImage/Notification` 的真实实现，**跑不起 app**，只能静态 `node --check`——查得出语法错，查不出"托盘空指针、菜单回调闭包失效、窗口事件时序、IPC 漏注册"这类只在运行时暴露的问题。所以每步都要在 Windows 上真起一次。
