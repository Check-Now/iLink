# Contributing

Thanks for helping improve iLink.

## Development

```powershell
npm install
npm run dev
```

Run checks before submitting changes:

```powershell
npm test
npm run build
```

## Scope

- Keep changes small and tied to the issue or bug being fixed.
- Do not commit `data/`, `data-2/`, `dist/`, `release/`, logs, or local credentials.
- Avoid changing networking, storage, or encryption behavior without tests.

## Pull Requests

Use a clear title and include:

- what changed
- why it changed
- how it was tested
